19 Reminder on the bias-variance tradeoff

  • Reminder: ML lingo uses the word “Variance” in a slightly different way than we’re used to:

    • Here, variance measures the difference in performance between on the training vs testing data

      • This is almost unrelated to the more familiar (to us) use of variance to describe a distribution.

20 One way to reduce variance: Shrinkage

  • Shrinkage estimators offer lower variance by construction

    • It’s literally just lowering your beta values.

    • The more we shrink coefficient estimates, the lower their variance across samples.

  • To build intuition on this, start with the extreme case: shrink all slope estimates to zero, make constant predictions.

    • Performance would be constant!

21 Quick example and implications for bias

22 Starting with OLS

  • When we have many observations, OLS does a good job

    • Why? Because with tons of observations, it is more likely the training data reflect other possible data.

      • E.g., suppose we have measurements of weight and size of mice. Fits pretty well.

23 But what if we have fewer data?

  • Suppose we only get to fit the data on a small subset (red)

    • The resulting estimate is not similar to the full data in green

24 Variance on out-of-sample observations is huge

  • Our predicted line will have minimum bias on the training data

    • It’s the “Best Linear Unbiased Estimate” (BLUE)

  • But, there is huge variance on green.

  • Main idea of regularization:

    • What if we shrink the COEFFICIENT value in the OLS regression

    • This will obviously introduce more bias

  • But, as we will see, this almost always reduces out-of-sample variance

    • This is also referred to as “shrinkage”

25

  • Why did we just do this?

  • In general (VERY VERY OFTEN) shrinking our coefficients will improve out-of-sample performance

    • It is so important that in big data and ML applications, almost everything is regularized in some way.

41 Many possible instruments in big data contexts

42 How many instruments?

  • Asymptotic theory:

    • IF all instruments are exogenous and relevant, then they contain useful information.

    • For the most efficient (low variance) estimator, we should use all available information. Question is just how to combine it.

    • 2SLS can help recover optimal combination of those instruments.

  • But, we often have finite samples

  • And, having too many instruments can introduce bias!

    • Hence, we need some well-thought-out model selection approach.

43 Correct for overfitting-bias with LASSO

  • If bias arises from overfitting the first stage, we could try to limit overfitting.

    • Belloni, Chen, Chernozhukov, Hansen (2012 Econometrica): LASSO!

  • Rough idea:

    • Estimate first stage via LASSO (with specific penalization scheme)

    • Use instruments selected via LASSO in first stage estimated via OLS

    • Use fitted values in second stage
